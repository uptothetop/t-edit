# Agent Instructions: SDET & Performance

Этот проект представляет собой высокопроизводительный, ультра-минималистичный Markdown-редактор для Windows 10, используемый на слабых устройствах с процессором Intel Atom x5-Z8550 и медленным eMMC. Основной приоритет: нулевая задержка ввода (typing latency) и низкое потребление ОЗУ.

Для соблюдения качества кода и производительности применяются следующие строгие правила (SDET):

## 1. Юнит-тесты (Unit Testing)
- **Обязательное покрытие**: Любое изменение в бизнес-логике (все операции редактирования в структуре `GapBuffer`) ОБЯЗАТЕЛЬНО должно сопровождаться созданием или обновлением юнит-тестов.
- **Фреймворк**: Используется легковесный однозаголовочный фреймворк `doctest`.
- **Pass condition**: Агент ОБЯЗАН запускать тесты локально на macOS перед тем, как считать задачу завершенной. Все тесты должны быть зелеными (100% pass rate).

## 2. Безопасность и проверка памяти (Security & Memory Safety)
- **Проверка памяти**: Код должен строго проверяться на утечки памяти (Memory Leaks) и выходы за пределы массива (Out-of-bounds/Buffer Overflow). Учитывая работу с сырыми буферами в `GapBuffer`, это критически важно.
- **Использование Sanitizers**: В сборку, предназначенную для локального тестирования, в обязательном порядке включаются санитайзеры: AddressSanitizer (ASAN) и UndefinedBehaviorSanitizer (UBSAN).
- **Флаги компилятора**: `-fsanitize=address,undefined` (также рекомендуется флаг `-g` для сохранения отладочных символов).

## 3. Тесты производительности (Performance Testing)
- **Целевая метрика**: Задержка ввода (latency) при редактировании текста.
- **Бенчмарк**: Должен быть реализован специальный тест производительности, который симулирует тяжелый файл.
- **Условия симуляции**: 
  - Начальный размер текста в `GapBuffer`: **1 МБ** (один мегабайт).
  - Выполнение операций изменения курсора, вставки (insert) и удаления (erase) символа по центру (или в случайном месте) буфера.
- **Условие провала (Fail condition)**: Если вставка или удаление **ОДНОГО** символа в `GapBuffer` занимает более **1 миллисекунды (1 ms)** на macOS, тест считается проваленным. Код должен работать за считанные микросекунды на быстром компьютере, чтобы обеспечить моментальный отзыв на целевом планшете.
